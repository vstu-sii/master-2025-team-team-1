## Use-Case: Автоматический анализ резюме и подготовка к интервью

**Actor:** HR-специалист (зарегистрированный и авторизованный)  
**Goal:** Получить структурированный список кандидатов с оценкой соответствия вакансии, выявленными навыками и пробелами, а также персонализированными вопросами для собеседования - за минимальное время.

---

## Общее описание сценария

| Пункт | Описание |
|-------|----------|
| **Назначение сценария** | Автоматизировать первичный отбор кандидатов: проанализировать резюме, ранжировать по соответствию вакансии, выявить ключевые навыки и пробелы, сгенерировать персонализированные вопросы для интервью. |
| **Предусловия** | - Установлено браузерное расширение AI HR Assistant<br>- Пользователь авторизован с аккаунта компании<br>- Пользователь находится на странице с резюме (hh, Habr Career и др.)<br>- Загружено или введено описание вакансии<br>- Страница с резюме полностью загружена |
| **Участники** | HR-специалист, Менеджер по найму |
| **Критерий успешности сценария** | - Резюме успешно спаршены и структурированы<br>- Сформирован рейтинг кандидатов (Top / Medium / Low)<br>- Вычислен скоринг соответствия (0-100%)<br>- Выявлены сильные стороны и пробелы у кандидатов<br>- Есть возможность сгенерировать 5-7 персонализированных вопросов<br>- Данные доступны для экспорта в CSV или Google Sheets |
| **Инициирующее событие** | Нажатие кнопки «Начать парсинг» или «Анализировать по вакансии» в панели расширения |

---

## Целевые сегменты пользователей

| Сегмент | Основная задача | Ключевая потребность |
|---------|----------------|-----------------------|
| HR-специалисты рекрутинговых агентств | Подбор большого количества кандидатов на разные позиции | Быстро отсеивать неподходящих и выделять топ-кандидатов |
| HR-отделы компаний среднего бизнеса | Массовый подбор персонала | Автоматизация рутинных задач: парсинг, сравнение, экспорт |
| Руководители команд / менеджеры по найму | Участие в собеседованиях | Получение готовых, релевантных вопросов без анализа резюме |

---

## Основные боли пользователей
- Высокие временные затраты на чтение и анализ десятков или сотен резюме.  
- Риск субъективной ошибки при отборе кандидатов.  
- Сложность в выявлении реальных навыков и пробелов на основе текста резюме.  
- Отсутствие единой системы обработки резюме в разных форматах (PDF, DOCX, текст).  
- Недостаток инструментов для подготовки содержательного собеседования на основе данных кандидата.  

---

## Основной сценарий (Happy Path)

1. HR-специалист переходит на сайт поиска работы (hh, Habr Career и др.) и открывает список кандидатов.  
2. Запускает панель расширения AI HR Assistant.  
3. Нажимает кнопку «Начать парсинг».  
4. Расширение извлекает данные всех видимых резюме: ФИО, опыт работы, навыки, зарплата, контакты.  
5. HR загружает описание вакансии (текст или файл).  
6. ИИ:  
   - Анализирует требования вакансии  
   - Сравнивает каждый профиль с требованиями  
   - Вычисляет скоринг соответствия (0-100%)  
   - Формирует рейтинг кандидатов: Top (≥80%), Medium (60-79%), Low (<60%)  
   - Определяет сильные стороны и пробелы  
   - Генерирует 5-7 персонализированных вопросов  
7. В правом нижнем углу появляется уведомление: «Спаршено 24 резюме».  
8. В интерфейсе расширения отображается:  
   - Общее количество обработанных резюме за день  
   - Кнопки экспорта (CSV, Google Sheets)  
   - Возможность просмотра анализа и вопросов по каждому кандидату  
9. Менеджер по найму получает ссылку или файл с вопросами и использует их при собеседовании.  

**Результат:** за 8–10 минут завершён первичный скрининг 20+ кандидатов с полным набором материалов для принятия решений.

## Альтернативные сценарии

| Сценарий | Описание |
|----------|----------|
| **A. Без описания вакансии** | Система выполняет только структуризацию резюме. Ранжирование и скоринг недоступны. Экспорт возможен. |
| **B. Парсинг одного резюме** | Анализ выполняется для одного кандидата. Сразу отображаются навыки, пробелы и вопросы. |
| **C. Изменение весов критериев** | HR корректирует приоритеты (например, повышает вес «опыта в управлении»). ИИ пересчитывает скоринг и обновляет рейтинг. |
| **D. Частичный экспорт** | Пользователь выбирает конкретных кандидатов → экспортирует только их в CSV или Google Sheets. |

---

## Исключительные сценарии (обработка ошибок)

| Ошибка | Реакция системы |
|--------|-----------------|
| Не найдено резюме на странице | Уведомление: «На этой странице не обнаружены резюме. Перейдите в раздел поиска кандидатов.» |
| Ошибка парсинга (антибот, JS-защита) | Подсказка: «Страница временно недоступна. Обновите страницу или отключите блокировщик контента.» |
| Неподдерживаемый формат файла | Сообщение: «Поддерживаются PDF, DOCX, TXT. Загрузите файл в корректном формате.» |
| Отсутствует соединение с AI API | Локальное сохранение данных. Уведомление: «Анализ временно недоступен. Данные будут обработаны при восстановлении соединения.» |
| Превышен лимит бесплатных запросов | Диалог: «Достигнут дневной лимит. Обновите тариф для продолжения.» |
| Низкое качество резюме (скан без текста) | Метка: «Качество данных низкое - требуется ручная проверка.» |

## Ценность для пользователя

| Пользователь | Основная ценность |
|--------------|-------------------|
| HR-специалисты агентств | Ускорение закрытия вакансий в 5-6 раз. Возможность массового отбора с минимальными усилиями. |
| HR-отделы компаний | Снижение нагрузки на 60%. Автоматизация рутины при массовом найме. |
| Менеджеры по найму | Готовые вопросы позволяют проводить глубокие собеседования без предварительного анализа резюме. |
| Компании в целом | Повышение качества найма, снижение текучести кадров, объективность отбора. |

**Уникальная ценность продукта:**  
AI HR Assistant экономит время HR-специалистов и повышает качество найма, автоматически анализируя резюме, ранжируя кандидатов и подготавливая персонализированные вопросы для интервью.
