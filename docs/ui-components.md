# Прототип интерфейсов

## 1. Ключевые Экраны (Wireframes)

### 1.1. Экран входа (Login/Main Entry)

**Назначение:** Аутентификация пользователя.
**Особенности:** Содержит поля "Email" и "Пароль", кнопку "Войти".
**Состояния:**
*   **Загрузка:** Кнопка "Войти" заблокирована со спиннером.
*   **Ошибка:** Сообщение "Неверный логин или пароль" красным цветом.

### 1.2. Dashboard / Список Вакансий

**Назначение:** Обзор и управление вакансиями.
**Особенности:** Боковая навигация (Sidebar), кнопка "Создать вакансию", интерактивная таблица с метриками (Средний скоринг, Кандидатов).
**Состояния:**
*   **Загрузка:** Таблица заменена скелетоном (Shimmer effect).
*   **Пусто:** Сообщение "У вас пока нет созданных вакансий" с призывом к действию.

### 1.3. Форма Создания/Редактирования Вакансии

**Назначение:** Ввод данных о вакансии и настройка скоринга.
**Особенности:**
*   **Основная информация:** Название, Описание, Зарплатный диапазон.
*   **Настройка Скоринга:** Специализированное поле для ввода **Ключевых навыков с весами** (например, `UML (x1.5)`), что напрямую влияет на ранжирование.
*   **Настройки парсинга:** Чекбоксы для автоматизации (Автоматическая прокрутка).

### 1.4. Детали Вакансии и Кандидаты

**Назначение:** Детальный просмотр и управление кандидатами по конкретной вакансии.
**Особенности:** Двухколоночный макет (Детали Вакансии + Список Кандидатов). Таблица кандидатов содержит **Скоринг (%)** и **Рейтинг (Top/Medium/Low)** с цветовым кодированием.

### 1.5. Всплывающее окно Расширения (Extension Popup)

**Назначение:** Управление парсингом на странице.
**Особенности:** Кнопка **"Начать парсинг"**, метрики ("Всего резюме"), опции экспорта данных (CSV, Google Sheets, Telegram).
**Состояния:**
*   **Ошибка:** Сообщение "Не удалось определить текущую страницу" (красный индикатор).
*   **Загрузка:** Кнопка "Начать парсинг" меняется на "Парсинг..." со спиннером.

## 2. Компонентная Архитектура UI

Интерфейс построен на модульной архитектуре, разделенной на три уровня:

| Уровень | Примеры Компонентов | Назначение |
| :--- | :--- | :--- |
| **Базовые (Atomic)** | Button, Input, Tag/Pill, Icon, Spinner. | Неделимые, переиспользуемые элементы. |
| **Молекулярные** | Form Field, Stat Card, Alert/Message, **Tags Input with Weight**. | Группы базовых элементов, выполняющие одну функцию. |
| **Органические** | Sidebar, Vacancy Form, Candidate Table Row, Extension Popup. | Сложные секции интерфейса, объединяющие молекулы. |

## 3. Responsive Design

Интерфейс адаптивен для Desktop и Mobile, с приоритетом на Desktop (основной сценарий HR).

| Элемент | Desktop (>= 1024px) | Mobile (< 768px) |
| :--- | :--- | :--- |
| **Layout** | Двухколоночный (Sidebar + Main Content). | Одноколоночный. Sidebar скрыт, доступен через "гамбургер-меню". |
| **Vacancy Details** | Две колонки (Детали + Кандидаты). | Колонки складываются в вертикальный стек. |
| **Таблицы** | Полная таблица. | Преобразование в **Card View** (карточки). |
