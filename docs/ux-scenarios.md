## UC-0: Авторизоваться в системе

**Цель:** Получить доступ к функционалу AI HR Assistant.
**Действующее лицо:** HR-специалист.

| Аспект | Описание |
| :--- | :--- |
| **Happy Path** | HR вводит корректные Email и Пароль, успешно входит в Dashboard. |
| **Edge Cases** | 1. Неверный пароль (3 попытки). 2. Учетная запись не активирована. 3. Сбой сервера авторизации. |
| **Точки LLM** | Отсутствуют. |
| **Микроинтеракции и Обратная связь** | **Валидация:** Кнопка "Войти" неактивна, пока поля не заполнены. **Обратная связь:** При ошибке - красный баннер с сообщением. **Успех:** Мгновенное перенаправление. |

**Цель:** Создать учетную запись для использования системы.
**Действующее лицо:** HR-специалист.

| Аспект | Описание |
| :--- | :--- |
| **Happy Path** | HR вводит валидные данные, подтверждает Email, успешно завершает регистрацию. |
| **Edge Cases** | 1. Попытка регистрации с уже существующим Email. 2. Несовпадение паролей. 3. Сбой отправки письма с подтверждением. |

## UC-2: Подобрать кандидатов по вакансии (Оркестрация)

**Цель:** Комплексный процесс поиска и оценки кандидатов на конкретную вакансию.
**Действующее лицо:** HR-специалист.

| Аспект | Описание |
| :--- | :--- |
| **Happy Path** | HR последовательно выполняет все шаги: создает вакансию, настраивает скоринг, запускает парсинг, просматривает результаты и, при необходимости, экспортирует их. |
| **Edge Cases** | 1. Сбой на любом из включенных шагов (UC-2.1 - UC-2.4) прерывает процесс. 2. HR пытается пропустить обязательный шаг (например, запустить поиск без создания вакансии). |
| **Точки LLM** | LLM используется в подпроцессах UC-2.3 (Скоринг) и UC-2.4 (Генерация вопросов). |
| **Микроинтеракции и Обратная связь** | **Прогресс:** На Dashboard отображается статус выполнения каждого подпроцесса (например, "Критерии заданы", "Поиск активен"). **Оркестрация:** Система блокирует переход к следующему шагу, пока не завершен предыдущий (например, нельзя запустить поиск, пока не создана вакансия). |

## UC-2.1: Создать вакансию

**Цель:** Внести информацию о новой вакансии в систему для дальнейшего подбора.
**Действующее лицо:** HR-специалист.

| Аспект | Описание |
| :--- | :--- |
| **Happy Path** | HR заполняет все поля формы, сохраняет, и вакансия появляется в списке активных. |
| **Edge Cases** | 1. HR пытается сохранить форму с незаполненными обязательными полями. 2. Ссылка на вакансию невалидна. 3. Сбой сохранения из-за ошибки БД. |
| **Точки LLM** | Отсутствуют. |
| **Микроинтеракции и Обратная связь** | **Валидация:** Мгновенная подсветка полей при ошибке. **Успех:** Перенаправление на страницу "Детали вакансии" или Dashboard. |

## UC-2.2: Уточнить критерии поиска кандидатов

**Цель:** Настроить кастомные критерии и их веса для скоринга резюме, чтобы обеспечить максимальную релевантность.
**Действующее лицо:** HR-специалист.

| Аспект | Описание |
| :--- | :--- |
| **Happy Path** | HR добавляет 3-5 ключевых навыков, устанавливает веса (x1.0 - x2.0) и успешно сохраняет настройки. |
| **Edge Cases** | 1. HR вводит нечисловое значение в поле "Вес". 2. HR пытается сохранить пустой список критериев. 3. Сбой сохранения из-за ошибки сети. |
| **Точки LLM** | Отсутствуют. (Настройки используются LLM в UC-2.3, но не генерируются им). |
| **Микроинтеракции и Обратная связь** | **Валидация:** Поле "Вес" принимает только числа. **Обратная связь:** При добавлении критерия он мгновенно появляется в списке. **Успех:** Toast-уведомление "Критерии сохранены". |

## UC-2.3: Запустить поиск кандидатов

**Цель:** Инициировать сбор данных резюме с внешнего ресурса и их автоматический скоринг.
**Действующее лицо:** HR-специалист.

| Аспект | Описание |
| :--- | :--- |
| **Happy Path** | HR запускает парсинг с опцией "Автопрокрутка", расширение собирает 50+ резюме, и все они успешно проходят скоринг. |
| **Edge Cases** | 1. **Ошибка парсинга:** Расширение не может определить структуру страницы. 2. **Сбой LLM:** Backend не получает ответ от LLM при запросе скоринга. 3. **Нет активной вакансии:** Парсинг не может быть запущен. |
| **Точки LLM** | **LLM-запрос:** Backend отправляет резюме (текст) и параметры скоринга (UC-2.2) в LLM для расчета процента соответствия и генерации краткого отчета. |
| **Микроинтеракции и Обратная связь** | **Загрузка:** Кнопка "Начать парсинг" меняется на "Парсинг..." со спиннером. **Прогресс:** Всплывающее окно отображает счетчик проанализированных резюме в реальном времени. **Ошибка:** Красный индикатор ошибки в расширении. |

## UC-2.4: Просмотреть и оценить найденных кандидатов

**Цель:** Визуально оценить результаты скоринга и получить дополнительную информацию для принятия решения о собеседовании.
**Действующее лицо:** HR-специалист.

| Аспект | Описание |
| :--- | :--- |
| **Happy Path** | HR просматривает список, находит кандидата с высоким скорингом, открывает полный отчет, просматривает сгенерированные LLM вопросы и копирует их. |
| **Edge Cases** | 1. **Нет данных:** Список кандидатов пуст. 2. **Сбой LLM:** Не удалось сгенерировать вопросы для интервью. 3. **Долгий ответ:** Генерация вопросов занимает более 10 секунд. |
| **Точки LLM** | **LLM-запрос:** Backend отправляет полный отчет о кандидате и описание вакансии в LLM для генерации 5-7 вопросов для интервью, основанных на пробелах и сильных сторонах. |
| **Микроинтеракции и Обратная связь** | **Визуализация:** Цветовое кодирование рейтинга (Top - зеленый, Low - красный). **Загрузка:** При генерации вопросов отображается спиннер в модальном окне. **Успех:** Кнопка "Копировать" рядом с вопросами. |

## UC-2.5: Экспортировать список кандидатов

**Цель:** Выгрузить проанализированные данные резюме для дальнейшей работы вне системы.
**Действующее лицо:** HR-специалист.

| Аспект | Описание |
| :--- | :--- |
| **Happy Path** | HR выбирает "Экспорт в Google Sheets", система, имея активную авторизацию, мгновенно создает таблицу и предоставляет ссылку. |
| **Edge Cases** | 1. **Сбой авторизации:** Не удалось подключиться к Google Sheets/Telegram. 2. **Большой объем данных:** Экспорт в CSV занимает слишком много времени (таймаут). 3. **Нет данных:** HR пытается экспортировать данные до завершения парсинга. |
| **Точки LLM** | Отсутствуют. |
| **Микроинтеракции и Обратная связь** | **Загрузка:** При экспорте в Google Sheets/Telegram отображается спиннер с текстом "Подготовка данных...". **Ошибка:** Модальное окно с сообщением "Ошибка авторизации Google. Пожалуйста, переподключите аккаунт." **Успех:** Toast-уведомление "Экспорт в CSV завершен. Файл загружен." |
