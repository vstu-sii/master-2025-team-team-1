# Roadmap проекта AI HR Assistant

## 1. Обзор проекта

**Продукт:** Браузерное расширение + веб-приложение для автоматизации первичного отбора кандидатов  
**Целевая аудитория:** HR-специалисты, менеджеры по найму среднего бизнеса  
**Длительность:** 7 спринтов (2 недели за спринт)  
**North Star Metric:** Сокращение времени на первичный отбор с 60 до ≤ 10 минут на 20 резюме

---

## 5. Управление рисками

### 5.1 Реестр рисков

| № | Риск | Вероятность | Влияние | Статус | План минимизации |
|---|------|-------------|--------|--------|-------------------|
| R1 | Низкая точность LLM при парсинге резюме | Высокая (70%) | Высокое | Active | Гибридный подход (regex + LLM), хорошие примеры в промпте, fallback на правила |
| R2 | Hallucination в генерируемых вопросах | Средняя (50%) | Среднее | Active | A/B тестирование, review человеком, penalization в промпте |
| R3 | Парсинг не работает на hh.ru после обновления | Средняя (40%) | Высокое | Active | Мониторинг DOM изменений, быстрая реакция, альтернативные методы |
| R4 | Масштабируемость: медленно при 100+ резюме | Средняя (50%) | Среднее | Active | Асинхронная обработка (Celery), кэширование (Redis), load testing в спринте 4 |
| R5 | Задержка в integrationen с Google Drive/Telegram | Низкая (20%) | Низкое | Monitor | Отложить на спринт 6, MVP может обойтись без этого |
| R6 | Члены команды недоступны / болеют | Низкая (30%) | Высокое | Mitigate | Документирование, cross-training, резервные роли |
| R7 | Бюджет на LLM API закончится | Средняя (40%) | Высокое | Active | Локальные модели как fallback, оптимизация промптов, кэширование |
| R8 | Chrome Extension не проходит审查 Web Store | Низкая (20%) | Среднее | Active | Ранняя подача на review (спринт 5), коммуникация с Google |
| R9 | GDPR требования не выполнены | Низкая (25%) | Высокое | Active | Consultancy с юристом, шифрование, data deletion flow, privacy policy |
| R10 | Нет time на полировку перед demo day | Средняя (50%) | Среднее | Monitor | Buffer в спринтах 5–6, приоритизация MVP функций |

### 5.2 План действий по рискам

#### R1: Низкая точность парсинга

**Минимизация:**
- Использовать гибридный подход: first-pass regex для извлечения структуры, then LLM для refinement
- Few-shot prompting с примерами хороших/плохих парсингов
- Fallback на ручной ввод пользователем

**Мониторинг:** Langfuse - метрика `parsing_accuracy` ≥ 85%

**Trigger:** Если accuracy < 80%, переход на baseline правила + LLM для сложных случаев

#### R2: Hallucination в вопросах

**Минимизация:**
- Constraint-based prompting (вопросы должны быть основаны только на резюме/вакансии)
- Human review на выборке вопросов (5% от всех)
- Явный penalty за hallucination в промпте

**Мониторинг:** Langfuse - метрика `hallucination_rate` ≤ 5%

**Trigger:** Если rate > 10%, A/B тест нового промпта

#### R3: Парсинг DOM hh.ru сломался

**Минимизация:**
- Мониторинг DOM структуры (регулярные проверки в CI)
- Альтернативные методы парсинга (Selenium, Puppeteer как fallback)
- Быстрая communication с пользователями при проблемах

**Мониторинг:** Автоматические E2E тесты на hh.ru каждый день

**Trigger:** Alert при failed parsing > 10% в день

#### R4: Медленная обработка при масштабе

**Минимизация:**
- Асинхронная обработка с Celery + RabbitMQ
- Redis кэширование результатов (24h TTL)
- Load testing в спринте 4 на 100+ резюме параллельно

**Мониторинг:** Prometheus метрика `processing_time_p99` ≤ 3 сек

**Trigger:** Если p99 > 5 сек, переход на async/batch обработку

#### R7: Бюджет на LLM API

**Минимизация:**
- Оптимизация промптов для уменьшения token usage
- Кэширование часто запрашиваемых анализов
- Использование более дешевых моделей (GPT-3.5 turbo вместо GPT-4)
- Plan local LLM deployment (Llama 2, Mistral) как fallback

**Мониторинг:** Dashboard с daily API costs

**Trigger:** Если monthly cost > budget на 20%, переход на оптимизацию

#### R6: Члены команды недоступны

**Минимизация:**
- Документирование всех критических процессов
- Cross-training (каждый knows 1-2 other role)
- Резервные люди для critical функций

**Мониторинг:** Еженедельные чек-ины здоровья команды
