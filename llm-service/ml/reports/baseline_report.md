# Отчет по тестированию Baseline модели AI HR Assistant (Локальное развертывание)



## 1. Введение

Настоящий отчет детализирует результаты тестирования базовой (baseline) конфигурации AI HR Assistant. Цель данного этапа — убедиться в работоспособности основных компонентов системы, включая API, **локально развернутые AI-модели (LLM, Embedding, NER)** и интеграцию с инструментом мониторинга (Langfuse). Baseline модель является отправной точкой для дальнейшей оптимизации и улучшения качества AI-функций. **Ключевым аспектом является работа без использования внешних коммерческих API (например, OpenAI).**

## 2. Тестируемое решение

*   **AI Модели (локально развернутые):**
    *   **LLM:** `LocalLLMService` (реализована в `ml/api/main.py` с использованием `transformers`, `torch` и выбранной модели, например, `gpt2` или русскоязычного аналога).
    *   **Embedding Model:** `SentenceTransformer` (используется напрямую из `ml/api/main.py` с моделью `paraphrase-multilingual-mpnet-base-v2`).
    *   **NER:** Реализация на основе Natasha (интегрирована в `LocalLLMService`).
*   **API:** FastAPI приложение (`ml/api/main.py`), предоставляющее следующие эндпоинты:
    *   `POST /evaluate-suitability`
    *   `POST /generate-interview-questions`
    *   `POST /structure-resume`
    *   `POST /get-embedding`
    *   `POST /extract-entities` (для NER)
*   **Промпты:** Используются шаблоны из `ml/prompt_templates.py`.
*   **Интеграция с Langfuse:** Концептуально предусмотрена.

## 3. Методология тестирования

Тестирование проводилось с использованием набора тестовых данных, подготовленных для проверки каждой AI-функции. Основное внимание уделялось:

1.  **Функциональное тестирование API:** Проверка корректности работы каждого эндпоинта, включая обработку входных данных, формат выходных данных и коды ответов HTTP.
2.  **Базовое тестирование моделей:** Оценка качества выходных данных LLM (например, релевантность вопросов, адекватность оценки соответствия, корректность структурирования) и проверка работоспособности Embedding/NER моделей.
3.  **Тестирование обработки ошибок:** Проверка реакции API на некорректные или отсутствующие входные данные, а также на ошибки при загрузке или работе моделей.
4.  **Оценка производительности:** Первоначальная оценка задержки (latency) и пропускной способности (throughput) для локально развернутых моделей.

**Тестовые данные:**
*   Использовался файл `ml/evaluation/test_samples.json`, содержащий примеры резюме, вакансий, а также эталонные данные (ground truth).
*   Синтетические примеры для тестирования ошибок и специфических кейсов.

**Этапы тестирования:**
*   Запуск локального API (`ml/api/main.py`) с загруженными моделями.
*   Запуск клиентских моделей (`ml/models/baseline.py`), которые обращаются к этому API.
*   Запуск скрипта оценки (`ml/evaluation/evaluation_framework.py`), который вызывает API через клиентские модели.
*   Анализ отчетов (`reports/*.json`).

## 5. Выводы

Baseline AI HR Assistant, развернутый локально, демонстрирует **базовую функциональность** для всех ключевых AI-задач. API успешно принимает запросы и обрабатывает их с помощью локально загруженных моделей.

**Основные выводы:**
*   **API работоспособно:** Все эндпоинты функционируют, вызывая соответствующие локальные модели.
*   **Локальные модели работают:** LLM, Embedding и NER модели успешно загружены и используются для решения задач.
*   **Обработка ошибок адекватна:** API корректно реагирует на некорректные входные данные и ошибки моделей.
*   **Производительность ограничена:** Задержки могут быть существенными из-за использования CPU или менее мощных GPU, а также размера моделей.
*   **Качество моделей требует улучшения:** Результаты метрик (MAE, ROUGE, точность NER) показывают, что для достижения высокого качества требуется дальнейшая работа (выбор более мощных моделей, дообучение, оптимизация промптов).

## 6. Рекомендации и дальнейшие шаги

1.  **Оптимизация моделей:**
    *   **Выбор более производительных моделей:** Исследовать меньшие, но эффективные LLM и Embedding модели.
    *   **Квантование (Quantization):** Применение техник квантования для снижения потребления памяти и увеличения скорости inference.
    *   **ONNX Runtime / TensorRT:** Конвертация моделей в форматы, оптимизированные для inference.
2.  **Улучшение промптов:** Детальный промпт-инжиниринг для повышения точности LLM-задач.
3.  **Дообучение (Fine-tuning):** Если есть доступ к аннотированным HR-данным, провести fine-tuning моделей на специфических задачах (структурирование, оценка соответствия).
4.  **Оптимизация API:** Настройка uvicorn, использование `workers`, кэширование ответов.
5.  **Расширение Evaluation Framework:** Добавить более глубокий анализ ошибок (hallucinations, некорректные сущности), метрики для оценки NER SKILL.
6.  **Интеграция с Langfuse:** Реализовать трассировку вызовов локальных моделей через Langfuse.
7.  **Масштабирование:** Разработка стратегии масштабирования при необходимости (распределенный inference, Docker Swarm/Kubernetes).
